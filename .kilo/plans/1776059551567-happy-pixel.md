# Voice-Flow Full App Improvement Plan

Based on live testing as `admin1@voiceflow.local` and deep codebase analysis.

---

## CRITICAL FIXES (Session-Breaking Bugs)

### 1. Middleware Not Wired — `proxy.ts` Never Executes
**Impact:** No server-side auth, no CSP headers, no redirect protection. Root cause of session loss on navigation.

`frontend/proxy.ts` exports `proxy()` instead of `middleware()`, and the file is named `proxy.ts` not `middleware.ts`. Next.js never loads it.

**Fix:** Create `frontend/middleware.ts`:
```ts
export { proxy as middleware, config } from './proxy';
```

**Files:** `frontend/middleware.ts` (new), `frontend/proxy.ts` (unchanged)

---

### 2. Session Cookie Race Condition
**Impact:** On navigation, `__session` cookie may not exist yet when middleware checks it, causing redirect to login.

`syncFirebaseSession()` is async (POST to `/api/auth/session`), but navigation can happen before it completes. The `onIdTokenChanged` listener fires the sync, but the cookie isn't guaranteed to be set before a hard navigation.

**Fix:** In `UserContext.tsx`, ensure `syncFirebaseSession()` completes before allowing protected navigation. Add a session-ready gate:
- Track `sessionCookieSynced: boolean` state
- Set to `true` only after successful `syncFirebaseSession()` 
- Prevent client-side navigation to protected routes until `sessionCookieSynced === true`
- Add `await syncFirebaseSession()` before `router.push()` calls in auth-dependent flows

**Files:** `frontend/contexts/UserContext.tsx`

---

### 3. Auth Gate Fallback Too Aggressive (350ms)
**Impact:** Users see a flash of the auth gate on every hard navigation because Firebase re-hydration takes 500ms-2000ms.

`AUTH_GATE_FALLBACK_MS = 350` in `WorkspaceRouteEntryScreen.tsx` shows the auth gate before Firebase has re-initialized the auth state from IndexedDB.

**Fix:**
- Increase `AUTH_GATE_FALLBACK_MS` from 350 to 2500ms
- Add a loading skeleton instead of the auth gate card for the initial load
- Check for existing `__session` cookie as a fast-path signal that the user is likely authenticated

**Files:** `frontend/src/app/workspace/WorkspaceRouteEntryScreen.tsx`

---

### 4. Vector Runtime Stuck in "Starting" State
**Impact:** Vector engine shows "Starting" indefinitely with no timeout or user feedback.

No timeout transitions the `'starting'` state to `'offline'`. If status polls fail, the state persists forever.

**Fix:**
- Add a 30-second timeout in `refreshTtsRuntimeStatus` that transitions `'starting'` → `'offline'`
- Show a toast: "Engine activation timed out. Try again or switch engines."
- Add a retry button on the runtime status UI when stuck in `'starting'`
- Fix `switchTtsEngineRuntime` default: don't default to `'starting'` when the response lacks a `state` field — instead, fetch status immediately

**Files:** `frontend/src/app/workspace/MainApp.tsx`, `frontend/services/mediaBackendService.ts`

---

## PERFORMANCE (Fast Play & Low Latency)

### 5. Audio Playback Preloading
**Impact:** No preloading of next audio chunks in live streaming mode. Each chunk starts decoding only after the previous one finishes.

**Fix:**
- In `AudioPlayer.tsx`, preload the next 2 live chunks when the current chunk starts playing (not when it ends)
- Use `decodeAudioData` on a background AudioContext to have the next buffer ready
- Preload the first 3 seconds of queued items when queue mode is active

**Files:** `frontend/components/AudioPlayer.tsx`

---

### 6. New AudioContext Per Mix Operation
**Impact:** `studioMixService.ts` creates a new `AudioContext` per `fetchTrackBuffer` call instead of reusing a singleton.

**Fix:** Use the singleton `getAudioContext()` pattern (already exists in `geminiService.ts` lines 499-516) in `studioMixService.ts` as well. Cache the music track buffer so it's not re-fetched on every mix.

**Files:** `frontend/services/studioMixService.ts`

---

### 7. Base64 Decode in Main Thread
**Impact:** `base64ToBlobUrl` in AudioPlayer decodes large audio chunks synchronously on the main thread, causing UI jank.

**Fix:**
- Move base64 → ArrayBuffer conversion to a Web Worker
- Or use `fetch(blobUrl)` pattern with `Response` to avoid manual decoding
- At minimum, break large chunks into smaller decode batches with `requestIdleCallback`

**Files:** `frontend/components/AudioPlayer.tsx`

---

### 8. Translation Cache
**Impact:** Every translation request hits the AI model. No caching of previous translations for the same text+language pair.

**Fix:**
- Add an in-memory LRU cache (max 50 entries) keyed by `hash(text) + targetLang`
- Cache entries expire after 30 minutes
- Show "Using cached translation" toast on cache hit

**Files:** `frontend/services/geminiService.ts`

---

### 9. Queue Audio Pre-generation
**Impact:** Queue items are processed sequentially with no lookahead. Users wait for each item one at a time.

**Fix:**
- When queue mode is active, pre-generate the next 2 items in parallel while the current item plays
- Add a `concurrency` setting to queue processing (default: 2)
- Show progress: "3/10 items ready • Generating item 4..."

**Files:** `frontend/src/features/studio/model/queue.ts`, `frontend/src/app/workspace/MainApp.tsx`

---

## SECURITY

### 10. CSP Headers Not Applied
**Impact:** Zero CSP protection in any environment because `proxy.ts` middleware never runs (see fix #1).

After wiring the middleware:
- Add `https://generativelanguage.googleapis.com` to `connect-src` (needed for direct Gemini API key mode)
- Add ngrok domains to `connect-src` in dev mode
- Test that all Firebase, Google, and Razorpay domains work under the CSP

**Files:** `frontend/proxy.ts`

---

### 11. Server-Side Rate Limiting is In-Memory Only
**Impact:** Rate limits reset on server restart and are inconsistent across horizontal instances.

**Fix:**
- Move rate limit counters to Redis (or Cloudflare KV/D1 for edge deployment)
- Keep in-memory as fallback if Redis is unavailable
- Add rate limit headers to responses: `X-RateLimit-Remaining`, `X-RateLimit-Reset`

**Files:** `frontend/src/shared/auth/apiAuthMiddleware.ts`

---

### 12. Add DOMPurify for XSS Defense-in-Depth
**Impact:** No explicit XSS sanitization. Relies solely on React escaping + CSP. If any `dangerouslySetInnerHTML` usage exists, it's vulnerable.

**Fix:**
- Install `dompurify` and `@types/dompurify`
- Create `sanitizeHtml(input: string): string` utility
- Audit all `dangerouslySetInnerHTML` usages and apply DOMPurify
- Use DOMPurify in the AI Director output rendering (script text may contain HTML-like content)

**Files:** New `frontend/src/shared/ui/sanitizeHtml.ts`, audit existing `dangerouslySetInnerHTML` usages

---

### 13. Admin Session Unlock Token Persistence
**Impact:** The `X-Admin-Unlock` token is in-memory only. If the user refreshes the page, the token is lost and must be re-issued.

**Fix:**
- Store the unlock token in `sessionStorage` (cleared on tab close, not on refresh)
- Auto-retrieve on admin panel load
- Add a 15-minute expiry with automatic re-prompt

**Files:** `frontend/src/shared/auth/adminAccess.ts`

---

## UX & SUPPORT

### 14. "Forgot Password?" Always Disabled
**Impact:** Button requires email to be typed first, but this isn't obvious to users. It also disables if Firebase is misconfigured.

**Fix:**
- Always enable the "Forgot password?" button
- On click without email: focus the email field and show "Enter your email first" hint
- If `firebaseIssue` is true, show "Password reset unavailable" with a support link instead of disabling

**Files:** `frontend/components/auth/Login.tsx`

---

### 15. `/app/account` Returns 404
**Impact:** No direct URL for account management. Users must navigate through the profile modal.

**Fix:**
- Add a route at `/app/account` that renders the `ProfileAccountView`
- Or add a redirect: `/app/account` → `/app` with the profile sheet open

**Files:** `frontend/app/(app)/app/account/page.tsx` (new)

---

### 16. Onboarding: No Progress Tracking or Skip
**Impact:** Single-page onboarding with no way to skip or track progress. Returning users see the same onboarding screen if their session expires.

**Fix:**
- Add a "Skip" button that sets `onboarding_dismissed` in localStorage
- Check `onboarding_dismissed` before showing onboarding
- Track onboarding completion in the user's Firestore profile

**Files:** `frontend/src/features/onboarding/Onboarding.tsx`

---

### 17. In-App Help Tooltips
**Impact:** Complex features (AI Director, Queue mode, Cast & Crew) have no inline help.

**Fix:**
- Add info icons next to complex controls that show tooltip explanations
- Key areas: AI Director mode selector, Queue toggle, Multi-Speaker toggle, Translate dropdown, Voice Clone "Add Ref"
- Content from existing prompt contracts and documentation

**Files:** `frontend/src/app/workspace/MainApp.tsx`, `frontend/components/` (tooltip component)

---

### 18. Connectivity-Aware UX
**Impact:** No visual indicator when the backend is unreachable. Users see generic errors.

**Fix:**
- Use the existing `connectivity.online`/`connectivity.offline` notification events to show a persistent banner
- Disable "Generate Audio" when offline with a tooltip explaining why
- Queue generation requests when offline and auto-execute when back online

**Files:** `frontend/src/app/workspace/MainApp.tsx`, `frontend/src/shared/notifications/`

## AUTO-ALLOCATION & INTELLIGENCE

### 19. Smart Engine Auto-Selection
**Impact:** Users must manually choose between VECTOR and PRIME. No guidance on which engine is better for their use case.

**Fix:**
- Auto-select VECTOR for single-speaker, short text (<1000 chars)
- Auto-select PRIME for multi-speaker scripts or text >3000 chars
- Show a "Recommended: PRIME" badge with reasoning
- Remember the user's last-used engine per mode (single/multi-speaker)

**Files:** `frontend/src/app/workspace/MainApp.tsx`, new `frontend/src/shared/workspace/engineAutoSelect.ts`

---

### 20. Auto Voice Assignment Improvements
**Impact:** Multi-speaker auto-voice uses gender heuristics only. No accent/age awareness in the default flow.

**Fix:**
- When script language is detected, prefer voices matching that language/region
- If a speaker name matches a known cultural pattern (e.g., Japanese name → Japanese voice), auto-assign the matching voice
- Remember user's per-speaker voice assignments across sessions (persist to Firestore)
- Add a "Lock voice" option per speaker to prevent auto-reassignment

**Files:** `frontend/src/app/workspace/MainApp.tsx`, `frontend/services/geminiService.ts`

---

### 21. Script Auto-Save & Recovery
**Impact:** No auto-save of the studio script. If the browser crashes, the script is lost.

**Fix:**
- Auto-save the current script to IndexedDB every 30 seconds (debounced)
- On studio load, check for unsaved drafts and offer recovery
- Show "Draft recovered" toast with option to dismiss or restore

**Files:** New `frontend/services/studioDraftService.ts`, `frontend/src/app/workspace/MainApp.tsx`

---

### 22. Generation Error Auto-Retry with Backoff
**Impact:** Failed generations show an error toast but don't auto-retry. Users must manually retry.

**Fix:**
- On transient errors (network, 429, 503), auto-retry up to 3 times with exponential backoff (2s, 4s, 8s)
- Show "Retrying... (2/3)" in the generate button
- On permanent errors (400, quota exceeded), show the error without retry
- Add retry count to queue items with auto-retry on transient failures

**Files:** `frontend/src/app/workspace/MainApp.tsx`, `frontend/src/features/studio/model/queue.ts`

---
