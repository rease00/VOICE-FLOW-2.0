# voice-Flow Cloudflare conversion

This checkout runs a Cloudflare-native frontend/backend split:

- `frontend/` is the React Router app deployed with Cloudflare Workers.
- `backend/` is the Hono worker used behind the frontend service binding.
- The root `server.mjs` is the local snapshot server used for quick browser verification.

## Local start

- Root snapshot server: `npm run dev` or `npm run start`
- Backend dev server on `8787`: `npm --prefix backend run dev`
- Backend Wrangler worker: `npm --prefix backend run dev:worker`
- Frontend app dev mode: `npm --prefix frontend run dev`

## Validation

- Root syntax check: `npm run check`
- Backend checks: `npm --prefix backend run check`
- Frontend types: `npm --prefix frontend run typecheck`
- Frontend production build: `npm --prefix frontend run build`
- Dependency audit: `npm audit --omit=dev`

## Deploy

- Backend worker: `npm --prefix backend run deploy`
- Frontend worker: `npm --prefix frontend run deploy`

## Config and secrets

- Backend runtime bindings live in `backend/wrangler.jsonc` and include D1, R2, queue, and Durable Object bindings.
- Frontend runtime bindings live in `frontend/wrangler.jsonc` and include the `BACKEND` service binding.
- `frontend/worker-configuration.d.ts` is generated by `wrangler types` during `npm --prefix frontend run typecheck` and is ignored by git.
- Keep real secrets out of `wrangler.jsonc`; use `wrangler secret put` for anything sensitive.

## Artifact hygiene

Generated output should stay untracked:

- `.wrangler/`
- `backend/.wrangler/`
- `frontend/.wrangler/`
- `frontend/build/`
- `frontend/.react-router/`
- `test-results/`
